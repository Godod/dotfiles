# Dunst Configuration

A highly customized and visually polished configuration for **Dunst**, a lightweight and customizable notification daemon for X11 and Wayland environments. This setup is fully integrated with the system's dynamic Material You theming engine.

## Overview

Dunst listens for notifications over D-Bus and displays them in a sleek, non-intrusive floating window. This configuration ensures that notifications are beautiful, responsive, and follow the active desktop color palette generated dynamically from your wallpaper.

### Key Features

- **Typography**: Uses `Google Sans Code 12` for clear and readable text layout.
- **Icon Set**: Complemented by the `Papirus-Dark` icon theme (located in `/usr/share/icons/Papirus-Dark/`).
- **Dynamic Theming**: Utilizes `matugen` to extract color keys from the wallpaper, writing them dynamically to `dunstrc` using custom templates.
- **Interactive Control**: Bindings for quick dismissal and interaction:
  - **Left Click**: Close current notification.
  - **Middle Click**: Perform notification action and close current.
  - **Right Click**: Dismiss all active notifications.
- **Waybar Integration**: A companion bash script (`notification.sh`) counts unread notifications in history and outputs Waybar-compliant JSON to display status icons.

## Tech Stack & Dependencies

- **Notification Daemon**: `dunst` (v1.9.0+)
- **Theme Engine**: `matugen` (for Material You dynamic coloring)
- **Icons**: `papirus-icon-theme`
- **Shell**: `bash` / `dunstctl` (for CLI controls and integration scripting)

## Configuration Details

### 1. Geometry & Layout
- **Monitor**: `0` (primary)
- **Position**: `top-right` with offset `(10, 20)`
- **Width**: Dynamic range from `0` to `300` pixels
- **Height**: Dynamic range from `0` to `300` pixels
- **Corner Radius**: `8px` for smooth rounded edges
- **Progress Bar**: Enabled at `10px` height for notifications displaying operations progress.

### 2. Urgency Definitions & Styling
Dunst classifies incoming notifications into three levels:

| Urgency | Timeout | Description / Color mappings |
|---|---|---|
| **Low** | `3s` | Background: Surface; Text: On Surface; Accent: Primary Container |
| **Normal** | `6s` | Background: Surface; Text: On Surface; Accent: Primary Container |
| **Critical** | Infinite (`0`) | Background: Surface; Text: On Surface; Accent: Error color (Red/Coral) |

## Dynamic Matugen Templates

The active configuration `dunstrc` is compiled from `config_template.toml`. The following token mapping is used by Matugen:

```toml
frame_color = "{{colors.primary.default.hex}}"
highlight = "{{colors.primary.default.hex}}"
separator_color= "{{colors.outline.default.hex}}"

[urgency_low]
background = "{{colors.surface.default.hex}}"
foreground = "{{colors.on_surface.default.hex}}"
highlight = "{{colors.primary_container.default.hex}}"
```

Whenever a new wallpaper is applied, `matugen` updates these fields and invokes `dunstctl reload` to apply the theme instantly without restarting the daemon.

## Available Scripts & Tools

### `notification.sh`
This script polls `dunstctl` to fetch notification counts and formatting for status bars:

```bash
# Get current history count
COUNT=$(dunstctl count history)
```

It outputs formatted JSON for Waybar:
- **Unread**: `{"text": " <span foreground='red'><sup></sup></span>", "tooltip": "X unread notifications"}`
- **Empty**: `{"text": "", "tooltip": "No notifications"}`

### Command-Line Interface (`dunstctl`)

Use these commands to interact with the Dunst service from the CLI or keybindings:

| Command | Action |
|---|---|
| `dunstctl close` | Close the most recent notification |
| `dunstctl close-all` | Close all active notifications |
| `dunstctl history-pop` | Re-open the last closed notification |
| `dunstctl set-paused toggle` | Toggle "Do Not Disturb" mode |
| `dunstctl count history` | Get the number of unread notifications |
| `dunstctl reload` | Force Dunst to reload the `dunstrc` file |

## Troubleshooting

### Testing Notifications
You can trigger dummy notifications to verify your styling and icons using `notify-send`:

```bash
# Test normal notification
notify-send "Test Notification" "This is a normal message with dunst styling."

# Test critical notification (remains open until dismissed)
notify-send -u critical "System Alert" "This is a critical notification!"
```

### Checking Daemon Logs
If notifications are not appearing, verify Dunst is listening to the system bus:

```bash
# Run Dunst in the foreground to view diagnostic prints
killall dunst && dunst -verbosity info
```

### Resetting Service
If the dynamic themes fail to refresh, reload Dunst manually:

```bash
dunstctl reload
```
